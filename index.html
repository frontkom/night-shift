In a nutshell

A nightly AI shift, one PR per task.

You pick the repos and the tasks. Night Shift runs each on its own schedule and opens a labelled pull request for every change.

01 · Configure

Pick repos and tasks once

Run /night-shift in a Claude Code session. A friendly wizard walks you through repos, tasks, and schedule.

→
02 · Run at night

Routines fire while you sleep

Claude Routines execute the picked tasks on the schedule. Default: weeknights, 23:00 UTC. Quiet weekends.

→
03 · Review at dawn

PRs labelled night-shift

Each task arrives as its own PR. gh pr list --label night-shift is the audit trail. Nothing auto-merges.

What it actually does

The tasks.

One PR per task. Labelled night-shift. Easy to ignore, easy to merge. Bundles group related tasks — turn each off per repo.

plans bundle · plans

Drives your plans and labelled issues to PRs

Drop a docs/*-PLAN.md, or tag an issue with night-shift (GitHub or Jira). Night Shift takes it from there.

  • build-planned-features — picks pending phases, opens a PR.
  • work-on-issues — GitHub Issues labelled night-shift.
  • work-on-jira-issues — same, but pulled from Jira.
docs bundle · docs

Four kinds of docs work — every week

Not just a changelog. Four separate tasks that keep your project legible.

  • update-changelog — entries for user-facing changes.
  • update-user-guide — refreshes READMEs and guides.
  • document-decisions — captures ADRs from history.
  • suggest-improvements — opinionated ideas to pitch.
tests bundle · code-fixes

Adds up to 10 tests per PR — your style

Looks at how you already write tests. Picks the highest-impact coverage gaps. Writes new tests that match your conventions and verifies they pass.

  • Targets uncovered branches, not just lines.
  • Mirrors your patterns (framework, naming, fixtures).
  • Caps at 10 tests so the PR stays reviewable.
a11y bundle · code-fixes

WCAG 2.1 AA audit. Fixed, in one PR

Audits key pages against contrast, keyboard nav, ARIA, labels, focus and headings. Fixes everything — and writes regression tests so it doesn't come back.

  • Contrast · keyboard · ARIA · labels · focus · headings.
  • Real markup fixes, not TODO: a11y stubs.
  • Adds regression tests against the fixed pages.
i18n bundle · code-fixes

Finds every hardcoded string

Whatever i18n framework you use, Night Shift sweeps the codebase, picks sensible keys, and lands them as a single PR.

  • Works with your existing framework — no migration.
  • Suggests semantic keys (checkout.cta.pay).
  • Marks new translations NEEDS_REVIEW.
audits bundle · audits

Five nightly audit sweeps

Each is its own task. Each opens its own focused PR — review or ignore areas independently.

  • find-security-issues — OWASP Top 10.
  • find-bugs — races, edge cases, dead paths.
  • improve-seo — metadata, OG, structured data.
  • improve-performance — bundle bloat, slow queries.
  • dep-audit — pins vulnerable transitive deps.
baseline bundle · code-fixes

Shrinks the lint baseline — one entry per night

That PHPStan / ESLint / Psalm / mypy / RuboCop baseline file with thousands of grandfathered suppressions? It shrinks now. Quietly. While you sleep.

  • Picks one suppressed violation.
  • Fixes the underlying code — not the baseline file.
  • Removes the suppression. Opens a PR.
flake bundle · audits

Hunts down flaky tests — one a night

Mines 14 days of CI history. Picks the worst recurring offender whose nondeterminism is obvious from the source. Fixes root cause — quarantine is a last resort.

  • Missing await, unseeded RNG, leaky state.
  • High bar — most nights produce zero PRs. That's the point.
  • Keeps red CI signals trustworthy.
auto-fix built-in · on every PR

Red CI? Reviewer comment? It fixes itself

Every Night Shift PR has autofix_on_pr_create on. The routine wakes up, investigates the failure or comment, and pushes a fix.

  • Red CI → reads the log, tries a fix, pushes.
  • Reviewer leaves a comment → answers in code.
  • You don't babysit nightly PRs.

FAQ

The honest questions.

Why run it at night?

Avoid multitasking hell. Agents and humans competing for the same files is chaos — give them different shifts and they cooperate instead of collide.

Don't occupy CI time. Overnight is the quietest CI window. Daytime CI stays fast for humans pushing real work.

And the real question: why should there be a single hour in the day where AI isn't working for us?

Does it work with my stack?

Yes — it works for all of them. Night Shift reads your repo the way a new developer would and follows the patterns it finds.

But now we need your help. Pick one task in the night-shift repo that would make your life easier — a framework-specific test convention, a translation quirk, a stack idiom — and ship a PR.

Heads up: new versions don't auto-update existing routines. Your routines stay frozen on whatever version you installed — re-install to pick up improvements.

Aren't automated AI commits risky?

Nothing auto-merges. Every change lands as a labelled pull request that a human has to review. If a PR is bad, you close it — same as any other PR. CI runs on every Night Shift PR, and your branch protection rules still apply. The agent doesn't bypass anything. Risk surface = your normal review pipeline.

What if a task isn't relevant for my project?

Turn it off per repo. The setup wizard runs a per-repo task picker — you choose which tasks fire for each project, and you can re-run /night-shift any time to change the selection.

No plan file? The plans task quietly skips. No baseline? Same. Night Shift is opinionated, not pushy.

Where do I start?

If you maintain a repo: pick one repo and add Night Shift today. That means you'll have a batch of PRs waiting for you on Monday morning.

By the end of next week, we want every active Frontkom repo on the night shift.

Nothing happens if you don't do it.
